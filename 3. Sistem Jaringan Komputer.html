Sistem Jaringan Komputer - Sistem Jaringan Komputer adalah perangkat komputasi yang saling terhubung, serta dapat bertukar data dan berbagi sumber daya satu sama lain - Tipe Arsitektur Jaringan Komputer : a. Arsitektur Client-server : Jaringan yang dirancang untuk pengguna akhir yg disebut client. untuk mengakses sumber daya dari komputer pusat yang disebut server *Kelebihan : - sistemnya terpusat, sehingga untuk membuat cadangan data lebih mudah - keamanan lebih baik - meningkatkan kecepatan berbagi sumber daya *Kekurangan : - Harganya Servernya Mahal - Membutuhkan administrator jaringan khusus untuk sumber daya b. Arsitektur Peer-To-Peer : Jaringan dimana semua komputer dihubungkan bersama dengan hak dan tanggung jawab yang sama untuk memproses data *Kelebihan : - Harga lebih murah dari client server - Jika ada komputer yang mati, maka komputer lain masih bisa bekerja - Maintenance lebih mudah, karena setiap komputer dikelola secara mandiri * Kekurangan : - Sulit membuat Cadangan data - Keamanan lebih buruk dari client server, karena perangkat di kelola sendiri - Jaringan komuter terbentuk dari 5 hal : a. Sender (pengirim) : perangkat yg memiliki data dan akan mengirimnya b. Receiver (Penerima) : perangkat yg menerima data c. Media Transmisi : Media penghubung untuk transfer data dari pengirim ke penerima Ex: Kabel, Gelombang Radio d. Protokol : Aturan yg disepakati oleh pengirim dan penerima e. Message : Data yang di transfer melalui jaringan komputer - Berdasarkan cakupannya jaringan komputer ada : a. LAN(Local Area Network) : Gedung dan Kantor. twisted pair dan koaksial b. MAN(Metropolitan Area Network): Kota atau area kampus yg luas. modem dan kabel c. WAN(Wide Area Network): Negara. saluran telepon, FO, satelite link d. PAN(Personal Area Network): jaringan yg diatur untuk 1 orang/Jarak 10 Meter. Laptop, ponsel, media player, e. SAN(Storage Area Network) : Jenis lain dari lan, yg digunakan untuk menangani transfer dan penyimpanan data yg besar f. VPN(Virtual private Network) : Jaringan pribadi untuk meningkatkan keamanan dan privasi saat akses jaringan asli. - Topologi jaringan : metode untuk menghubungkan 2 komputer atau lebih dengan kabel ataupun tanpa kabel sebagai media transmisi - Contoh Topologi Jaringan : a. Topologi Mesh : setiap perangkat terhubung ke perangkat lain melalui saluran tertentu. Protokol yg digunakan AHCP dan DHCP *Kelebihan : - Susunan andal - Kesalahan dapat di diagnosa dengan mudah *Kekurangan : - instalasi dan konfigurasinya sulit - banyak kabel yg dibutuhkan b. Topologi Star : semua perangkat terhubung ke hub atau swtich melalui kabel tunggal *Kelebihan : - Jika ada kendala pada 1 komputer tidak mempengaruhi komputer lainnya *Kekurangan : - Jika switch / hub rusak, maka jaringan akan terhenti - Kinerja jaringan sangat bergantung pada hub c. Topologi Bus : Setiap perangkat terhubung ke satu kabel saja dan terhubung ke backbone *Kelebihan : - mudah di implementasikan - mudah menambah atau menghapus perangkat *Kekurangan : - Jika ada gangguan pada kabel, maka seluruh jaringan akan terganggu - Jika traffic padat, sering terjadi collision/tabrakan data d. Topologi Ring : setiap perangkat terhubung dalam satu jaringan dengan bentuk lingkaran. *Kelebihan : - kemungkinan collision sangat kecil, karena satu arah - biaya pemasangan cukup rendah *Kekurangan : - jika ada kendala jaringan sulit untuk di perbaiki - penambahan/pengurangan perangkat baru dapat mempengaruhi seluruh perangkat e. Topologi Tree : Setiap perangkat saling terhubung dengan struktur hierarkies berbentuk pohon *Kelebihan : - Memungkinkan banyak perangkat terhubung - memungkinkan jaringan di isolasi *Kekurangan : - biaya pemasangan tinggi - jika hub pusat mengalami kendala, maka menggangu seluruh jaringannya f. Topologi Hybrid : Gabungan dari beberapa teknologi di atas, contoh star dengan Ring
